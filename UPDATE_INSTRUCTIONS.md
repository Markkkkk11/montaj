# Инструкция по обновлению после перезапуска

## Что было сделано

Завершён **Этап 2: Система заказов**. Реализована полная функциональность работы с заказами, откликами и интеграция с Яндекс.Картами.

## Шаги для запуска

### 1. Применить миграции базы данных

```bash
cd backend
npx prisma generate
npx prisma migrate dev --name add_orders_and_responses
```

### 2. Убедиться, что Docker запущен

```bash
docker-compose up -d
```

### 3. Проверить работоспособность

```bash
./test-system.sh
```

### 4. Запустить приложение

```bash
npm run dev
```

## Тестирование функционала

### Как заказчик:
1. Зарегистрируйтесь с ролью CUSTOMER
2. Перейдите в `/orders/create` и создайте заказ
3. Просмотрите заказы в `/orders`
4. Посмотрите карту в `/orders/map`

### Как исполнитель:
1. Зарегистрируйтесь с ролью EXECUTOR
2. Заполните профиль в `/profile`
3. Просмотрите доступные заказы в `/orders`
4. Откликнитесь на заказ
5. Проверьте списание комиссии в дашборде

## Важные файлы

- `STAGE2_COMPLETE.md` - детальное описание реализованного
- `MAPS_SETUP.md` - инструкция по настройке Яндекс.Карт
- `TEST.md` - инструкция по запуску тестов

## Возможные проблемы

1. **Ошибка миграции:** Убедитесь, что PostgreSQL запущен в Docker
2. **Тесты не проходят:** Проверьте, что Redis запущен
3. **Карта не работает:** Нужен API-ключ Яндекс.Карт (см. MAPS_SETUP.md)
